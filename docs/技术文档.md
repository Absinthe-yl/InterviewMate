# InterviewMate 技术文档

## 一、项目概述

### 1.1 项目简介

InterviewMate 是一个 AI 面试模拟平台，旨在帮助用户更好地准备技术面试。当前版本（V1）实现了用户认证模块，为后续功能扩展奠定基础。

### 1.2 功能范围

| 功能 | 状态 | 说明 |
|------|------|------|
| 用户注册 | ✅ 已完成 | 用户名、密码、昵称 |
| 用户登录 | ✅ 已完成 | JWT Token 认证 |
| 用户登出 | ✅ 已完成 | Token 黑名单机制 |
| 用户信息获取 | ✅ 已完成 | 获取当前登录用户信息 |
| 模拟面试 | 🔲 待开发 | AI 驱动的面试对话 |
| 知识库 | 🔲 待开发 | 八股文、算法等知识整理 |

## 四、数据库设计

### 4.1 用户表 (user)

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码(BCrypt加密)',
    nickname VARCHAR(50) COMMENT '昵称',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

### 4.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | BIGINT | 是 | 主键，自增 |
| username | VARCHAR(50) | 是 | 用户名，唯一 |
| password | VARCHAR(100) | 是 | 密码，BCrypt 加密存储 |
| nickname | VARCHAR(50) | 否 | 昵称，默认同用户名 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

## 五、API 接口文档

### 5.1 接口规范

#### 基础路径
```
http://localhost:8080/api
```

#### 统一响应格式
```json
{
  "code": 0,          // 0 成功，非 0 失败
  "message": "success",
  "data": {}          // 响应数据
}
```

### 5.2 认证接口

#### 用户注册

**请求**
```
POST /api/auth/register
Content-Type: application/json
```

**参数**
```json
{
  "username": "testuser",
  "password": "123456",
  "nickname": "测试用户"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，3-50 字符 |
| password | string | 是 | 密码，6-100 字符 |
| nickname | string | 否 | 昵称 |

**响应**
```json
{
  "code": 0,
  "message": "注册成功",
  "data": null
}
```

**错误码**
| code | message |
|------|---------|
| 1 | 用户名已存在 |
| 1 | 参数验证失败 |

---

#### 用户登录

**请求**
```
POST /api/auth/login
Content-Type: application/json
```

**参数**
```json
{
  "username": "testuser",
  "password": "123456"
}
```

**响应**
```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

**错误码**
| code | message |
|------|---------|
| 1 | 用户名或密码错误 |
| 1 | 用户不存在 |

---

#### 用户登出

**请求**
```
POST /api/auth/logout
Authorization: Bearer {token}
```

**响应**
```json
{
  "code": 0,
  "message": "登出成功",
  "data": null
}
```

---

### 5.3 用户接口

#### 获取当前用户信息

**请求**
```
GET /api/user/me
Authorization: Bearer {token}
```

**响应**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "username": "testuser",
    "nickname": "测试用户"
  }
}
```

**错误码**
| code | message |
|------|---------|
| 1 | 用户不存在 |
| 401 | 未授权（Token 无效或过期） |

## 六、安全设计

### 6.1 密码安全

- 使用 BCrypt 算法加密存储
- 加密强度：默认 10 轮
- 明文密码不落库、不落日志

```java
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
String hashedPassword = encoder.encode(rawPassword);
```

### 6.2 JWT Token 设计

#### Token 结构
```
Header.Payload.Signature
```

#### Payload 内容
```json
{
  "sub": "1",              // 用户ID
  "username": "testuser",  // 用户名
  "iat": 1713945600,       // 签发时间
  "exp": 1713952800        // 过期时间
}
```

#### Token 配置
| 配置项 | 值 | 说明 |
|--------|-----|------|
| 算法 | HS256 | HMAC SHA-256 |
| 过期时间 | 2 小时 | 7200000ms |
| 存储位置 | Redis | 支持主动失效 |

### 6.3 Token 黑名单机制

登出时将 Token 加入 Redis 黑名单：

```
Key: token:{token_value}
Value: blacklisted
TTL: 2 小时
```

### 6.4 请求认证流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  Client  │────▶│JwtFilter │────▶│  Redis   │────▶│Controller│
└──────────┘     └──────────┘     └──────────┘     └──────────┘
                      │
                      ▼
               1. 提取 Token
               2. 验证签名
               3. 检查黑名单
               4. 加载用户信息
               5. 设置 SecurityContext
```

## 七、配置说明

### 7.1 后端配置 (application.yml)

```yaml
server:
  port: 8080

spring:
  application:
    name: interviewmate
  datasource:
    url: jdbc:mysql://localhost:3306/interviewmate?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
    username: root
    password: ${DB_PASSWORD:root}
    driver-class-name: com.mysql.cj.jdbc.Driver
  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD:}

jwt:
  secret: ${JWT_SECRET:interviewmate-secret-key-must-be-at-least-256-bits-long-for-hs256}
  expiration: 7200000  # 2小时

mybatis:
  mapper-locations: classpath:mapper/*.xml
  configuration:
    map-underscore-to-camel-case: true
```

### 7.2 环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| DB_PASSWORD | root | MySQL 密码 |
| REDIS_HOST | localhost | Redis 主机 |
| REDIS_PORT | 6379 | Redis 端口 |
| REDIS_PASSWORD | - | Redis 密码 |
| JWT_SECRET | - | JWT 签名密钥 |

### 7.3 前端配置 (vite.config.js)

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  plugins: [vue()],

    port: 5173,

        changeOrigin: true

    }
  }
})
```

## 八、部署说明

### 8.1 环境要求

| 组件 | 版本要求 |
|------|----------|
| JDK | 17+ |
| Node.js | 18+ |
| MySQL | 8.0+ |
| Redis | 6.0+ |
| Maven | 3.8+ |

### 8.2 后端部署

```bash
# 1. 初始化数据库
mysql -u root -p < src/main/resources/db/init.sql

# 2. 编译打包
mvn clean package -DskipTests

# 3. 运行
java -jar target/InterviewMate-1.0.0.jar

# 或使用环境变量
java -DDB_PASSWORD=your_password -jar target/InterviewMate-1.0.0.jar
```

### 8.3 前端部署

```bash
# 1. 安装依赖
cd interviewmate-web
npm install

# 2. 开发环境运行
npm run dev

# 3. 生产环境构建
npm run build

# 4. 预览构建结果
npm run preview
```

### 8.4 生产环境建议

1. **HTTPS**: 必须启用 HTTPS
2. **JWT 密钥**: 使用强随机密钥，通过环境变量注入
3. **数据库密码**: 不要使用默认密码
4. **Redis 密码**: 生产环境建议设置密码
5. **跨域**: 配置允许的域名白名单
