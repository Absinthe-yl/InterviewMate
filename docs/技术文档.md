# InterviewMate 技术文档

## 一、项目概述

### 1.1 项目简介

InterviewMate 是一个 AI 面试模拟平台，旨在帮助用户更好地准备技术面试。系统集成了 AI 智能面试、知识库管理、用户认证等核心功能。

### 1.2 功能范围

| 功能 | 状态 | 说明 |
|------|------|------|
| 用户注册 | ✅ 已完成 | 用户名、密码、昵称 |
| 用户登录 | ✅ 已完成 | JWT Token 认证 |
| 用户登出 | ✅ 已完成 | Token 黑名单机制 |
| 用户信息获取 | ✅ 已完成 | 获取当前登录用户信息 |
| 用户审核 | ✅ 已完成 | 管理员审核新用户注册 |
| AI 模拟面试 | ✅ 已完成 | AI 驱动的面试对话 |
| 面试记录与回放 | ✅ 已完成 | 查看历史面试记录 |
| 面试报告生成 | ✅ 已完成 | AI 生成面试评估报告 |
| 知识库管理 | ✅ 已完成 | 八股文、技术知识点整理 |
| Prompt 模板管理 | ✅ 已完成 | 管理员可编辑 AI Prompt |
| 岗位智能推断 | ✅ 已完成 | AI 从简历自动推断岗位 |
| 知识库关联提问 | ✅ 已完成 | 根据简历技能匹配知识库 |

## 四、数据库设计

### 4.1 用户表 (user)

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码(BCrypt加密)',
    nickname VARCHAR(50) COMMENT '昵称',
    role VARCHAR(20) DEFAULT 'USER' COMMENT '角色(USER/ADMIN)',
    status VARCHAR(20) DEFAULT 'PENDING' COMMENT '状态(PENDING/APPROVED/REJECTED)',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

### 4.2 面试会话表 (interview_session)

```sql
CREATE TABLE interview_session (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    resume_file_name VARCHAR(255),
    resume_data TEXT,
    position_type VARCHAR(50),
    difficulty_level VARCHAR(20),
    total_rounds INT DEFAULT 5,
    current_round INT DEFAULT 0,
    status VARCHAR(20) DEFAULT 'IN_PROGRESS',
    total_score DECIMAL(5,2),
    started_at DATETIME,
    ended_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='面试会话表';
```

### 4.3 面试轮次表 (interview_round)

```sql
CREATE TABLE interview_round (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id BIGINT NOT NULL,
    round_number INT NOT NULL,
    question_category VARCHAR(50),
    question TEXT,
    user_answer TEXT,
    score DECIMAL(5,2),
    feedback TEXT,
    ai_evaluation TEXT,
    status VARCHAR(20) DEFAULT 'ASKING',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_session_id (session_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='面试轮次表';
```

### 4.4 面试报告表 (interview_report)

```sql
CREATE TABLE interview_report (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id BIGINT NOT NULL UNIQUE,
    strengths TEXT,
    weaknesses TEXT,
    suggestions TEXT,
    overall_comment TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_session_id (session_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='面试报告表';
```

### 4.5 知识库表 (knowledge)

```sql
CREATE TABLE knowledge (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    content LONGTEXT,
    summary VARCHAR(500),
    category_id BIGINT,
    tags VARCHAR(500),
    file_name VARCHAR(255),
    view_count INT DEFAULT 0,
    status TINYINT DEFAULT 1,
    created_by BIGINT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='知识库表';
```

### 4.6 分类表 (category)

```sql
CREATE TABLE category (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL UNIQUE,
    description VARCHAR(200),
    sort_order INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='分类表';
```

### 4.7 Prompt模板表 (prompt_template)

```sql
CREATE TABLE prompt_template (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    template_key VARCHAR(50) NOT NULL UNIQUE COMMENT '模板标识',
    template_name VARCHAR(100) NOT NULL COMMENT '模板名称',
    template_content TEXT NOT NULL COMMENT '模板内容，支持{变量}占位符',
    description VARCHAR(500) COMMENT '模板说明',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_template_key (template_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Prompt模板表';
```

**预置 Prompt 模板：**

| template_key | 说明 |
|--------------|------|
| POSITION_INFER | 从简历推断岗位类型 |
| INTERVIEW_QUESTION | 生成面试问题 |
| ANSWER_EVALUATE | 评价候选人回答 |
| INTERVIEW_REPORT | 生成面试报告 |

### 5.2 认证接口

**请求**
```

Content-Type: application/json
```

**参数**
```json
{
  "username": "testuser",
  "password": "123456",
  "nickname": "测试用户"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|

| password | string | 是 | 密码，6-100 字符 |
| nickname | string | 否 | 昵称 |

**响应**
```json

  "code": 0,

### 5.3 面试接口

**请求**
```
POST /api/interview/start
Content-Type: multipart/form-data
Authorization: Bearer {token}

**参数**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| resume | File | 否 | 简历文件 (PDF/Word/TXT) |
| positionType | String | 否 | 岗位类型（不传则 AI 自动推断） |
| difficulty | String | 否 | 难度 (EASY/MEDIUM/HARD)，默认 MEDIUM |

**响应**
```json
{
  "code": 0,
  "data": {
    "sessionId": 1,
    "firstQuestion": "请介绍一下你的项目经验...",

```

## 七、安全设计

### 7.1 密码安全

- 使用 BCrypt 算法加密存储
- 加密强度：默认 10 轮
- 明文密码不落库、不落日志

```java
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
String hashedPassword = encoder.encode(rawPassword);
```

### 7.2 JWT Token 设计

#### Token 结构
```
Header.Payload.Signature
```

#### Payload 内容
```json
{
  "sub": "1",              // 用户ID
  "username": "testuser",  // 用户名
  "iat": 1713945600,       // 签发时间
  "exp": 1713952800        // 过期时间
}
```

#### Token 配置
| 配置项 | 值 | 说明 |
|--------|-----|------|
| 算法 | HS256 | HMAC SHA-256 |
| 过期时间 | 2 小时 | 7200000ms |
| 存储位置 | Redis | 支持主动失效 |

### 7.3 Token 黑名单机制

登出时将 Token 加入 Redis 黑名单：

```
Key: token:{token_value}
Value: blacklisted
TTL: 2 小时
```

### 7.4 权限控制

| 角色 | 权限 |
|------|------|
| USER | 面试、知识库查看 |
| ADMIN | 用户审核、Prompt 管理、知识库管理 |

## 九、部署说明

### 9.1 环境要求

| 组件 | 版本要求 |
|------|----------|
| JDK | 17+ |
| Node.js | 18+ |
| MySQL | 8.0+ |
| Redis | 6.0+ |
| Maven | 3.8+ |

### 9.2 后端部署

```bash
# 1. 初始化数据库
mysql -u root -p < src/main/resources/db/init.sql

# 2. 编译打包
mvn clean package -DskipTests

# 3. 运行
java -jar target/InterviewMate-1.0.0.jar

# 或使用环境变量
java -DDB_PASSWORD=your_password -DAI_API_KEY=your_key -jar target/InterviewMate-1.0.0.jar
```

### 9.3 前端部署

```bash
# 1. 安装依赖
cd interviewmate-web
npm install

# 2. 开发环境运行
npm run dev

# 3. 生产环境构建
npm run build

# 4. 预览构建结果
npm run preview
```

### 9.4 生产环境建议

1. **HTTPS**: 必须启用 HTTPS
2. **JWT 密钥**: 使用强随机密钥，通过环境变量注入
3. **数据库密码**: 不要使用默认密码
4. **Redis 密码**: 生产环境建议设置密码
5. **AI API 密钥**: 通过环境变量注入，不要硬编码
6. **跨域**: 配置允许的域名白名单
